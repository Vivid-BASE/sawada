# ウェブサイトコンテンツの更新方法

## 概要
このウェブサイトのコンテンツは、Googleスプレッドシートを編集するだけで簡単に更新できます。

## 日常的な更新手順

### 1. Googleスプレッドシートを開く
管理者から共有されたGoogleスプレッドシート「Sawada Website Content」を開きます。

### 2. 内容を編集する
更新したいタブ(Schedule、Discography、Projectsなど)を選択し、内容を編集します。

**できること:**
- ✅ 新しい行を追加(新しいイベントやディスコグラフィーを追加)
- ✅ 既存の行を編集(日付、タイトル、説明などを変更)
- ✅ 行を削除(不要なコンテンツを削除)

**注意点:**
- ⚠️ **1行目(ヘッダー行)は編集しないでください**
- ⚠️ 列の順番を変更しないでください
- ⚠️ タブ名を変更しないでください

### 3. 保存する
Googleスプレッドシートは自動保存されます。特別な操作は不要です。

### 4. 待つ
- **自動更新**: 毎時0分に自動的に同期されます(最大1時間待ち)
- **手動更新**: すぐに反映したい場合は、管理者に連絡してGitHub Actionsから手動実行してもらいます

### 5. 確認する
ウェブサイト(https://vivid-base.github.io/sawada/)を開いて、変更が反映されているか確認します。

## 各タブの編集方法

### Scheduleタブ(スケジュール)
新しいイベントを追加する場合:
1. 最終行の下に新しい行を追加
2. 各列に情報を入力:
   - **date**: 日付(例: 2024.03.15)
   - **place**: 場所(例: 東京文化会館)
   - **title**: タイトル
   - **description**: 説明
   - **image**: 画像パス(例: /images/event1.jpg)

### Discographyタブ(ディスコグラフィー)
新しいCDを追加する場合:
1. 最終行の下に新しい行を追加
2. 各列に情報を入力
3. **couplings**列: カンマ区切りで曲名を入力(例: 曲1,曲2,曲3)
4. **links**列: JSON形式で入力(例: {"amazon":"URL","tower":"URL"})

### Projectsタブ(プロジェクト)
新しいプロジェクトまたはイベントを追加する場合:
1. 同じプロジェクトの複数イベントは、**projectId**を同じにします
2. 各イベントごとに1行追加します

### PastEventsタブ(過去のイベント)
過去のイベントを追加する場合:
1. 最終行の下に新しい行を追加
2. 各列に情報を入力

### Profileタブ(プロフィール)
プロフィール情報を更新する場合:
1. 該当する**field**の行の**value**列を編集

### Historyタブ(ヒストリー)
年表を更新する場合:
1. 新しい年を追加する場合は新しい行を追加
2. **events**列: カンマ区切りでイベントを入力(例: イベント1,イベント2,イベント3)

## よくある質問

**Q: 編集後、いつウェブサイトに反映されますか?**
A: 毎時0分に自動同期されます。すぐに反映したい場合は管理者に連絡してください。

**Q: 画像はどうやって追加しますか?**
A: 画像ファイルは管理者にアップロードを依頼してください。画像パスをスプレッドシートに記入します。

**Q: 間違えて編集してしまいました**
A: Googleスプレッドシートの「編集履歴」から以前のバージョンに戻せます。または管理者に連絡してください。

**Q: 新しいタブを追加できますか?**
A: 既存のタブのみ編集可能です。新しいタブが必要な場合は管理者に相談してください。
